## Overview

### The Challenge

Users should be able to:

- View the optimal layout for the site based on the provided design images.
- See a fully structured and styled web page using HTML and CSS.

The project emphasizes:

- Building sections one at a time, starting with HTML then CSS.
- Utilizing Flexbox for layout.
- Implementing fundamental CSS concepts like `margin`, `padding`, `box-sizing`, and external font integration.
- Practicing good Git and GitHub workflow (committing early & often).
- Not worrying about pixel-perfection or mobile responsiveness at this stage.

## My Process

This project was approached by breaking down the design into distinct sections (Header, Hero, etc.) and building each one sequentially. For each section, I focused on completing the HTML content first, followed by the corresponding CSS styling.

### Built With

- **HTML5:** For the page structure.
- **CSS3:** For all styling, including:
  - Flexbox for layout.
  - Universal CSS Reset (`margin: 0; padding: 0; box-sizing: border-box;`).
  - Google Fonts (Roboto).
- **Git & GitHub:** For version control and project hosting.
- **VS Code:** As the development environment.

### What I Learned

During this project, I reinforced my understanding of:

- Setting up a robust project foundation with a CSS reset and `box-sizing: border-box`.
- Integrating external fonts (Google Fonts) into a web page.
- Building complex layouts using Flexbox, managing spacing with `justify-content` and `align-items`.
- Structuring a web page logically based on a visual design.
- The importance of committing early and often with clear, descriptive commit messages.
- The workflow of pushing local changes to a remote GitHub repository.

### Continued Development

In future projects, I aim to focus on:

- Improving responsiveness for various screen sizes (mobile-first approach).
- Exploring more advanced CSS properties and selectors.
- Potentially adding interactive elements using JavaScript (once I learn the basics).
